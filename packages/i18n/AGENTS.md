# @commercetools-uikit/i18n

## Purpose

Contains all translation messages for UI Kit components, published so consumers
can load locale data for internationalized component strings.

## How To Work Here

- Source messages live in each component's `messages.ts` file, not here.
- `data/core.json` is the extraction target — regenerated by `yarn extract-intl`
  from all component `messages.ts` files.
- Locale files (`data/de.json`, `data/es.json`, etc.) contain translations keyed
  by message ID. Add translations manually to these files.
- Run `yarn compile-intl` to produce the `compiled-data/` output that consumers
  load at runtime.

## Gotchas

- Do not hand-edit `data/core.json` — it is overwritten by `yarn extract-intl`.
- `compiled-data/` is generated output. Run `yarn compile-intl` after changing
  any `data/*.json` file.
- The `presets/ui-kit` package copies `data/` into its own `i18n/` directory
  via its `copy-assets` script — run it explicitly to propagate changes.
